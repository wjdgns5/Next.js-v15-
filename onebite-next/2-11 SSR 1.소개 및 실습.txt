서버 사이드 렌더링
SSR - Server Side Rendering

1. 가장 기본적인 사전 렌더링 방식
2. 요청이 들어올 때 마다 사전 렌더링 진행 함
---------
/pages/index.tsx 파일
export const getServerSideProps = () => {};

SSR 사전 렌더링 방식으로 이루어진다.
getServerSideProps 약속된 이름을 만들어서 내보내주었기 때문이다.

getServerSideProps이 SSR 방식으로 Home() 페이지 컴포넌트 보다 먼저 실행이 되어서 
getServerSideProps 인덱스 페이지에 필요한 데이터나 또 다른 백엔드 서버에서 불러오는 기능이다.

쉽게 정리하면
localhost:3000 << 인덱스 페이지 경로로 요청이 들어오게 되면
getServerSideProps << SSR이 동작을 해서 백엔드 서버로부터 데이터를 불러온다.
Home() << 컴포넌트가 실행되게 된다.
-----------------------------------

SSR은 반드시 객체를 반환해야 한다.
객체는 return으로 객체 타입인 props를 반환해야 한다.

------------------------------------

Home() 컴포넌트 또한 서버에서 한번 실행한 후에 (사전 렌더링 방식으로 서버에서 실행)
그 다음에 브라우저에서 한번 더 실행한 다음에  (브라우저 자바 Bundle 형식으로 실행이 될 때, 즉 Hydration 작업이 실행될때)
--------
Home() 컴포넌트 안에 window.location; 같은 것은 브라우저에서 실행되는거지 서버에서 실행되지는 않는다.
Home() 컴포넌트 안에 있더라도 처음 localhost:3000를 실행하게 되면 서버에서도 한 번 실행하기 때문에 오류가 발생한다.
Server Error - "ReferenceError: window is not defined" 이게 발생한다.
------------ 
그렇지만  localhost:3000를 실행하더라도 window함수를 사용해야 겠다 싶으면 useEffect(); 를 사용하면
페이지가 로딩되었을때 작동하거나 또는 특정 상황에서 작동하게 사용할수 있게 하는 기능이다.
(화면에 나타난 이후 작동하도록 설정되어 있는 함수. 서버에서는 실행되지 않는다.)

---------------

InferGetServerSidePropsType<> 
서버사이드 함수의 반환 타입을 자동 추론해서 페이지 컴포넌트에 타입을 적용

InferGetServerSidePropsType<typeof getServerSideProps>
getServerSideProps --> 데이터를 서버에서 불러오는 함수
